### LLM 无状态性与上下文管理
- **LLM 是无状态的**：每次 API 调用都是一次独立的前向传播，模型本身没有记忆
- **messages 数组 = 唯一的外部记忆**：开发者通过 messages 数组手动管理上下文
- **三种消息角色**：`system`（行为约束）、`user`（用户输入）、`assistant`（模型回复）
- **Context Window**：每个模型有最大输入长度限制，messages 不能无限增长
- **Token 随对话增长**：每轮把历史消息全文发回，prompt_tokens 逐轮增大

## 关键理解

> **LLM 开发的核心心智模型**：模型是无状态的 → messages 数组是唯一的外部记忆 → 开发者自己管理上下文生命周期。这决定了 Token 成本、上下文窗口限制、记忆系统设计等一切上层问题。

> **学习方法论关键调整**：程序性知识（SDK/API 调用）不适合纯苏格拉底提问——需要"示范→模仿→修改→独立"的渐进路径。I Do → We Do → You Do。
