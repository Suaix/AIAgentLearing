### 2. LLM API 调用基础（动手实践）
- **第一个 API 调用**：使用 OpenAI SDK 兼容接口调用 DeepSeek API
- **API 请求结构理解**：
  - `model`：指定模型（deepseek-v4-flash / deepseek-v4-pro）
  - `messages`：对话消息数组，包含 `system`、`user`、`assistant` 三种角色
  - `max_tokens`：限制输出 token 数
- **API 响应结构理解**：
  - `choices[0].message.content`：模型回复正文
  - `usage.prompt_tokens` / `completion_tokens` / `total_tokens`：Token 用量
  - `reasoning_content`：推理模型的内部思考过程

### 3. LLM 核心概念
- **Token**：LLM 处理文本的基本单位，不是字符/单词
  - 实测：英文约 5.8 字符/token，中文约 1.7 字符/token（中文 token 化效率较低）
- **Temperature**：控制输出随机性（0.0 确定性 → 1.0+ 创意性）
  - 本质是调整 next-token 概率分布的平滑程度
- **System Prompt**：设定 AI 角色和行为规则，是 Agent 开发的关键控制手段
- **流式输出（Streaming）**：逐 token 返回，提升用户体验
- **推理模型特性**：DeepSeek V4 会在回答前进行内部推理（reasoning_content），推理 token 计入 `max_tokens` 配额——设 `max_tokens` 太小会导致回答被截断

## 遇到的问题与解决

1. **Homebrew 安装需要 sudo**：沙箱环境无法交互式输入密码，需要在终端手动执行
2. **Python 版本问题**：macOS 系统自带 3.9.6，不符合 3.12+ 要求 → 通过 Homebrew 安装 3.12.13
3. **uv 默认找到系统 Python**：需要 `--python` 参数指定 Homebrew 版本
4. **httpx 代理冲突**：`~/.zshrc` 中配置了 `http_proxy`/`https_proxy`，httpx 自动使用但格式不兼容 → `httpx.Client(trust_env=False)` 绕过
5. **推理模型输出为空**：DeepSeek V4 的 `reasoning_content` 消耗 `max_tokens` 配额 → `max_tokens` 需设置足够大（建议 ≥300）

## 关键理解

> **LLM 本质**：给定上文，预测下一个 token 的概率分布，然后采样。这个简单的机制在足够大的模型和训练数据下涌现出推理、翻译、编程等复杂能力。

> **Agent 与普通 LLM 调用的区别**：普通调用是"一问一答"，Agent 则在循环中自动调用工具、观察结果、反思修正，直到完成任务。今天学的 API 调用是 Agent 的"基础设施"。
