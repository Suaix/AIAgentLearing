### 2. 单轮 vs 多轮 Tool 调用（Step 0~4 完整流程）
- **核心认知**：Agent 跑几轮不由代码分支决定，而由 LLM 看到工具结果后自主判断——返回 `tool_calls` 继续循环，返回纯文本结束
- **单轮**：LLM 在发出工具调用前已掌握全部所需信息
- **多轮**：后续工具调用的参数依赖前面工具的结果，必须先看到中间结果才能决定下一步
- **三种多轮驱动因素**：数据依赖、条件判断、失败重试
- **并行条件**：同一轮内互不依赖的工具可以并行调用
- **max_turns**：多轮调用的安全阀，过小导致截断，过大浪费成本

## 遇到的问题与解决

1. **Python 中文引号嵌套导致 SyntaxError**：`print("... "中文引号" ...")` 中，文件内的中文双引号被 Python 解析为 ASCII 双引号。解决：外层用单引号 `print('...')` 或使用「」替代中文引号。

2. **工具函数返回 dict 而非 JSON 字符串**：`get_restaurants` 直接返回 Python 字典 `{city: result}`，导致 API 报 `content should be a string` 400 错误。根因：`run_agent` 将工具返回值直接作为消息 content 传给 API，必须是字符串。修复：`return json.dumps(...)`。

3. **Python 嵌套函数的作用域疑问**：函数内部定义的函数，通过引用传递（字典、参数）可以在外部被调用——Python 函数是"一等公民"。
