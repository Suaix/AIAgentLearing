### 1. JSON Schema 基础 & 跨平台差异（知识补充）
- 学习了 JSON Schema 的核心思想：用 JSON 写"数据说明书"，声明数据应有什么字段、什么类型、哪些必填
- 掌握 Tool Use 中最常用的 6 个 JSON Schema 关键字：`type`、`properties`、`required`、`description`、`enum`、`default`
- 对比了 OpenAI / Anthropic / Gemini 三家平台的工具 Schema 差异（字段名、包装结构、类型大小写、响应格式）
- 了解了 MCP 作为跨平台标准化方向

### 2. 单轮 vs 多轮 Tool 调用（Step 0~4）
- **核心认知**：Agent 跑几轮不由代码分支决定，而由 LLM 看到工具结果后自主判断——返回 `tool_calls` 继续循环，返回纯文本结束
- **单轮**：LLM 在发出工具调用前已掌握全部所需信息
- **多轮**：后续工具调用的参数依赖前面工具的结果
- **三种多轮驱动因素**：数据依赖、条件判断、失败重试
- **并行条件**：同一轮内互不依赖的工具可以并行调用

### 3. Tool 选择策略 — tool_choice & parallel_tool_calls（Step 0~4）
- **tool_choice 四种模式**：auto（自主）、required（强制）、none（禁止）、指定工具名（精确控制）
- **核心发现**：约束越紧，LLM 越可能"扭曲"用户意图来适配强制工具
- **parallel_tool_calls**：控制同一轮内是否允许并行，DeepSeek 不完全支持此参数

## 遇到的问题与解决

1. **Python 中文引号嵌套导致 SyntaxError**：`print("... "中文引号" ...")` 中中文双引号被解析为 ASCII。解决：外层用单引号。
2. **工具函数返回 dict 而非 JSON 字符串**：`get_restaurants` 返回 Python 字典导致 API 400 错误。修复：`return json.dumps(...)`。
3. **Python 嵌套函数作用域**：函数内部定义的函数，通过引用传递可被外部调用——Python 函数是"一等公民"。
4. **DeepSeek thinking mode 与 tool_choice 不兼容**：使用 `tool_choice="required"/"none"` 时需关闭 thinking mode。
