# 学习记录 — 2026-06-30

## 今日学习内容

### Transformer 架构 & Self-Attention 机制
- **Self-Attention 核心机制**：每个 token 通过 Q/K/V 三个投影，直接"看到"序列中所有其他 token，生成融合上下文的新向量表示
- **Q/K/V 三元组**：Query（我想找什么）→ Key（我是什么标签）→ Value（我的实际内容），三个分开的设计让"匹配"和"传递"各司其职
- **Scaled Dot-Product Attention**：`softmax(Q·K^T / √d_k) · V`，缩放因子 `/√d_k` 防止大维度下点积值过大
- **Multi-Head Attention**：并行多组 Q/K/V（通常 8 头），不同头学习不同语言模式（语法、语义、位置、指代）
- **完整 Transformer 层**：Self-Attention（横向通信）+ Feed-Forward（纵向加工），交替 N 层
- **Temperature 本质**：temperature 调的是 softmax 的"锐度"——低温度→注意力尖峰化；高温度→均匀分散

### 关键认知
- Attention 计算量 O(n²) → 这就是 Context Window 不能无限大的物理原因
- 开发者不手写 Attention 层，但理解它影响 Prompt 设计（关键信息放哪里影响注意力流向）
- `exp()` 函数对输入极端敏感 → 数值稳定化（减 max、除以 √d_k）不是可选的，是必须的

## 遇到的问题与解决

1. **numpy 找不到**：`uv run python` 使用 `.venv` 虚拟环境，但 `pyproject.toml` 未声明 numpy 依赖。解决：`uv add numpy`。
   - 教训：本项目所有 Python 代码用 `uv run python` 运行，编写新代码前检查依赖是否在 pyproject.toml 中声明。
2. **实验③大数值不溢出**：[100, 200, 300] 不溢出，因为 exp(300) ≈ 1.9e130 在 float64 范围。改为 [800, 900, 1000] 后才溢出。float64 上限约 exp(709)。
