## 各维度评分

### 维度 A：概念理解深度 — L2 🟡（2.0 分）

**评估依据**：

| 来源 | 具体证据 |
|------|---------|
| Step 1 苏格拉底 | 每日记录中正确总结了 Q/K/V 三元组的分工（"Query 我想找什么 → Key 我是什么标签 → Value 我的实际内容"）。对 Scaled Dot-Product 公式中 `/√d_k` 的作用理解正确（"防止大维度下点积值过大→softmax 变极端→梯度消失"）。 |
| Step 4 费曼复述 | 一句话总结："Self-Attention 让序列中每个 token 直接看到所有其他 token，生成一个融合了上下文信息的新向量表示。Transformer = Self-Attention（横向通信）+ Feed-Forward（纵向加工）交替 N 层。" — 精准覆盖了核心机制。 |
| 笔记深度 | 笔记中主动讨论了"为什么 Q/K/V 要分开设计"（"如果 Q=K=V 直接用原始 X，模型只能按语义相似度关注，学不到'追'关注'猫'这种不相似但相关的关系"）— 说明有深层理解。 |

**达到该等级的理由**：

基础概念（Q/K/V 分工、Scaled Dot-Product、Multi-Head、Transformer 层结构）理解正确且能用自己的话表述。但学习者自评"掌握程度一般"，且笔记中部分深层解释（如"Q/K/V 分开让匹配和传递各自独立优化"）更像是复述教学材料而非内化后的自主表达。从"知道"到"能独立讲解给他人"之间还有距离。

### 维度 B：代码实践能力 — L2 🟡（2.0 分）

**评估依据**：

| 来源 | 具体证据 |
|------|---------|
| Step 0 热身 | 成功运行 `attention_warmup_v2.py`，观察到了手工权重产生的有意义的注意力模式（动词"追"关注主语"猫"和宾语"老鼠"）。 |
| Step 3-1 改参数 | Temperature 公式补全正确（`scores / (np.sqrt(d_k) * temp)`），三个温度值的输出验证了预期（temp↓→尖峰化，temp↑→均匀化）。 |
| Step 3-2 加功能 | Head 2 的 5 行计算代码全部正确补全，两个 Head 的注意力模式对比输出正常。 |
| Step 3-3 修 Bug | 正确识别了 unsafe_softmax 的溢出问题。额外发现了 [100, 200, 300] 不溢出的原因（float64 上限约 exp(709)），超出教学预期。 |
| Step 3-4 扩展场景 | 新句子的 embedding 设计完成，但「学生」和「书」的 embedding 完全相同（`[0.9, 0.1, 0.1, 0.1, 0.1]`），说明理解了"名词→名词性维度高"的模式，但未考虑"不同名词应有不同语义表示"的细微差别。 |

**达到该等级的理由**：

所有 4 个实验均正确完成，代码无误。Step 3-3 中自主探索了 float64 溢出边界，表现超出预期。但 experiment 4 的 embedding 设计缺乏区分度，且整体上更多是"按脚手架填空"而非"独立设计实验"——代码能跑通，但自主探索的主动性有限。

## 薄弱点清单

| # | 薄弱环节 | 所属维度 | 严重程度 | 建议行动 |
|---|---------|---------|---------|---------|
| 1 | Q/K/V 分开设计的深层原因能复述但未必能独立推演——从"知道答案"到"能从第一原理推导"之间还有距离 | A | 中 | 一周回检时尝试不看笔记，向自己解释"为什么不能直接用 X 做 attention？为什么必须是三个不同的投影？" |
| 2 | Embedding 设计时"学生"和"书"用了相同的向量——说明对语义空间的理解停留在词性层面，未到语义层面 | B | 低 | 回顾 experiment 4，思考"学生"和"书"应该在哪些维度上有差异（如：情感、力度？），重新设计更有区分度的 embedding |
| 3 | 代码实践整体偏"跟随脚手架"——能正确填空但自主探索（如尝试额外的温度值、设计第 3 个 Head）的主动性不足 | B | 中 | 后续 Step 3 任务中，完成基础 TODO 后主动尝试 1 个额外变体，培养"跑通了→还能怎么改"的习惯 |
| 4 | 笔记的"开发启示"与代码实验的绑定不够紧密——Temperature 实验直接对应 API temperature 参数的选择策略，但笔记中这个连接是隐含的 | C | 低 | 在笔记的"开发启示"中显式引用实验编号（如"实验①演示了 temperature↓→注意力尖峰化 → 所以创意写作用高 temperature，代码生成用低 temperature"） |

**强弱模式总结**：

知识结构化（L3）明显强于概念理解（L2）和代码实践（L2）。学习者善于整理和组织知识，但将知识内化到"能随时调用和独立应用"的程度还需要更多练习。这是一个典型的"笔记写得好但心里没底"的模式——笔记的完整度和理解深度之间还有落差。

## 复习建议

### 短期（本周内）
- 重读笔记中"为什么需要 Self-Attention"和"Q/K/V 三元组"两节，重点关注意义而非公式
- 回到 `attention_step3.py`，尝试 experiment 4 的变体：设计第 3 个句子（如"老师 讲 课"），从头设计 embedding，不看原始代码

### 长期（后续模块中）
- **模块 2（Prompt Engineering）**：学到这里时回看"开发启示 1"——Prompt 结构决定注意力流向，检验自己能否用 attention 知识解释"为什么 system prompt 比 user message 更稳定"
- **模块 5（Agent 架构）**：Multi-Head = 多视角 → 多 Agent = 多角色，检验这个类比是否成立
- **模块 9（生产化）**：数值稳定性（softmax 溢出）知识会在这里再次出现

### 回检计划
- **建议回检日期**：2026-07-14 前后（学习日 + 2 周）
- **回检方式**：合上笔记，口头解释"Self-Attention 的完整计算过程，从 token embedding 到输出向量"
- **判定标准**：能说清 6 个步骤（embedding → Q/K/V 投影 → 点积分数 → 缩放 → softmax → 加权求和），且能解释"除以 √d_k"和"减去 max(x)"两个数值技巧的原因
