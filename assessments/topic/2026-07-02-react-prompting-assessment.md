## 各维度评分

### 维度 A：概念理解深度 — L3 🟢（3.0 分）

**评估依据**：

| 来源 | 具体证据 |
|------|---------|
| Step 1 苏格拉底 | 问题 1：准确数出 4 个 ReAct 循环（查北京→查深圳→算倍数→给答案）。问题 2：一针见血指出"标准 Prompt 靠训练记忆，不可能获取 2188 万"。问题 3：精准区分 ReAct（工具调用获取外部数据）和 CoT（纯靠训练数据推理）。 |
| Step 4 费曼复述 | 第一轮：Thought 正确归因于模型。Action/Observation 归属有混淆（说"都由工具完成"），经纠正后理解。追问：正确回答了"Action + Final Answer 同时出现时继续执行 Action"——理解了循环优先级。 |
| 三层对比 | 能清晰区分 Standard（纯记忆）→ CoT（慢推理，仍靠记忆）→ ReAct（推理+工具，真实数据）三者的递进关系。 |

**达到该等级的理由**：

对 ReAct 循环的四段式结构（Thought → Action → Observation → repeat）和三层对比（Standard/CoT/ReAct）有清晰理解。Action/Observation 的"谁做决策 vs 谁执行"有一次混淆，但纠正后掌握了"模型决策、代码执行"的分离概念。元认知：理解了 Step 0 中 Task 1 模型不按格式输出的原因是"模型在没有真正工具执行环境时会模拟完整输出"。

### 维度 B：代码实践能力 — L3 🟢（3.0 分）

**评估依据**：

| 来源 | 具体证据 |
|------|---------|
| Step 0 热身 | 成功运行 `react_warmup.py`，观察到实验 C 中 Task 1 格式问题（模型输出 JSON 而非 ReAct 格式）和 Task 3 的多轮工具调用成功。 |
| Step 3-① 改参数 | max_turns=5 vs 1 对比实验完成。max_turns=1 只完成了一步（查北京），理解安全阀含义。 |
| Step 3-② 加工具 | **四个位置全部正确完成**：KNOWLEDGE_BASE_WEATHER（3 城天气数据）→ get_weather()（查询逻辑）→ System Prompt（声明工具+格式）→ 正则/路由（`search_city\|calculate\|get_weather`）。测试输出显示 Agent 正确调用了 get_weather[北京] 和 get_weather[深圳]，并给出有依据的对比分析。 |
| Step 3-③ 修 Bug | 准确定位到 `serch_city` 拼写错误（少 'a'），理解"模型输出正确但正则不匹配 → 静默失败 → 格式错误死循环"的因果链。 |
| Step 3-④ 扩展场景 | 餐厅推荐 Agent 四步法完成（数据→函数→Prompt→循环）。`search_restaurant()` 有一个小缺陷：for 循环中 `return` 在第一个匹配就返回，不返回全部匹配结果。当前 DB 每个菜系只有一家，不影响运行，但多结果场景会漏数据。 |

**达到该等级的理由**：

4 个任务全部独立完成，任务 ② 的"给 Agent 加工具四步法"掌握扎实——数据源→函数→Prompt→正则/路由，每一步都正确。任务 ④ `search_restaurant` 的单结果返回是唯一的技术缺陷，学习者认识到并选择接受（当前场景不影响）。核心的 ReAct 模式迁移能力已建立。

## 特殊记录

### 教学事故与修复

本次学习中出现两次教学事故：

1. **Step 3 首次生成时 TODO 被自行填满**：Claude 在首次生成 `react_step3.py` 时将所有 TODO 位置写入了完整实现，导致学习者的动手环节被跳过。学习者及时指出，文件被重写为真正的骨架格式。**已写入 CLAUDE.md 硬约束规则和持久记忆**，防止复发。

2. **Step 3 运行方式不友好**：四个任务在一个文件中通过 `input()` 串联，不符合学习者的交互偏好（命令行参数控制）。**CLAUDE.md 已更新**，固定 Step 3 运行方式为 `python xxx_step3.py {1-4}`。

### Action/Observation 概念混淆

Step 4 费曼检验中，学习者将 Action 和 Observation 都归为"由工具完成"。这是一个常见的概念混淆点——Action 是**模型决策 + 代码执行**的分离过程。经即时纠正后，学习者在追问中展现了正确理解（"会继续执行 Action"），说明混淆已被澄清。

- **模块 7（MCP）**：对比手动工具注册（四步法）和 MCP 的标准化工具发现机制

### 回检计划
