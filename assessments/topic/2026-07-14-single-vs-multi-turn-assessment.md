## 薄弱点与复习建议

1. **三种多轮驱动因素的独立识别**：当前对"数据依赖"的掌握最牢固，"条件判断"和"失败重试"是通过 Step 2 讲解和任务③意外发现的。后续在模块 5 Agent 架构中，留意这三种模式在不同架构中的表现差异。
2. **工具返回格式的一致性**：任务②的 dict/json 混淆虽然是 Python 语法层面问题，但在实际工程中（多团队协作、第三方 API 对接）是常见陷阱。建议在后续实践中始终统一为 `return json.dumps(...)` 模式。
