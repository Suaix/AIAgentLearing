## 维度 D：长期保持与迁移 — L3 🟢（3.0 分）

> 此维度基于 Step 5 独立里程碑项目「Token-Aware 对话 Agent」评估。

**评估依据**：

| 指标 | 证据 | 评价 |
|------|------|------|
| 独立构建能力 | 从零写了一个完整的对话 Agent，未参考 `code/01-foundations/` 下任何代码 | ✅ 真正独立 |
| 知识点整合 | 单文件整合了 4 个模块 1 核心概念：Token计量 + Context管理 + Agent四要素循环 + Temperature调控 | ✅ 跨主题融会贯通 |
| 问题自我定位 | 费用异常时自己发现（对比预期），Context 截断从"直接退出"迭代到"pop旧消息"，选型从 `break` 到消息条数阈值 | ✅ 能独立迭代 |
| 代码质量 | 结构清晰、变量命名合理、API 调用正确。存在 Context 截断循环触发的小瑕疵（pop 后 len 仍在阈值边缘），但不影响核心功能。 | 🟡 小瑕疵 |
| 迁移能力 | 模型中只有 DeepSeek 的 API Key，但 Agent 架构（system prompt + messages 数组 + 循环）与模型无关——换成 Claude/GPT 只需改 client 配置，代码主体不动。 | ✅ 架构解耦 |

**达到 L3 的理由**：

从零独立构建了一个可运行的多轮对话 Agent，正确整合了模块 1 的 4 个核心概念，且代码架构（messages 数组管理 + OpenAI 统一接口）天然支持模型切换。2 轮迭代中能自己发现问题方向并改进（成本单位、截断策略）。未达 L4 的唯一原因是 Context 截断的边界处理不够健壮，以及缺少错误处理（API 调用失败时没有 try/except）。
